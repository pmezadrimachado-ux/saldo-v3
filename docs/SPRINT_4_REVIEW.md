# Sprint 4 — Onboarding Inicial

## Status

Concluída.

## Entregas

- Detecção de onboarding pendente.
- Redirecionamento automático para `#/onboarding`.
- Tela de primeiro uso.
- Seleção de contas/cartões sugeridos.
- Campo de conta personalizada.
- Seleção de categorias de despesa.
- Campo de categoria personalizada.
- Seleção de categorias de receita.
- Conclusão do onboarding.
- Persistência de contas no IndexedDB.
- Persistência de categorias no IndexedDB.
- Atualização de settings.
- Atualização de preferences.
- Redirecionamento para `#/quick-add` após finalizar.
- Router protegendo rotas antes do onboarding.

## Critérios de aceite

- Primeiro acesso abre onboarding.
- Usuário consegue concluir configuração inicial.
- App cria pelo menos uma conta/cartão.
- App cria categorias de despesa e receita.
- Dados persistem ao recarregar.
- Onboarding não aparece novamente após concluído.
- App abre em Lançar após conclusão.
- Imports, exports, sintaxe e assets validados.
